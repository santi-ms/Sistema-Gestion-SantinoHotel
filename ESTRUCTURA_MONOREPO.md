# 📦 Estructura de Monorepo - Hotel Santino

## 🎯 Propuesta de Estructura

```
HotelGestion2/                    # Monorepo principal
├── backend/                      # Backend (FastAPI)
│   └── hotel-santino-backend/
│       ├── hotel.py
│       ├── requirements.txt
│       ├── README.md
│       └── ...
│
├── frontend/                     # Frontend (React/Vue/Next.js)
│   └── hotel-frontend/
│       ├── src/
│       ├── package.json
│       └── ...
│
├── docs/                         # Documentación compartida
│   ├── DEPLOYMENT.md
│   ├── ENDPOINTS.md
│   ├── CONFIGURACION_VARIABLES.md
│   └── ...
│
├── .gitignore                    # Git ignore unificado
├── README.md                     # README principal del monorepo
└── (archivos de configuración compartidos)
```

## ✅ Ventajas

- ✅ Puedo analizar frontend y backend juntos
- ✅ Cambios coordinados entre ambos
- ✅ Documentación centralizada
- ✅ Un solo repositorio Git
- ✅ Más fácil mantener sincronizado

## 🔄 Pasos para Organizar

¿Quieres que organice esto ahora? Puedo:

1. **Crear la estructura de carpetas**
2. **Mover el backend a `backend/hotel-santino-backend/`**
3. **Mover el frontend a `frontend/hotel-frontend/`** (cuando me indiques la ruta)
4. **Organizar la documentación en `docs/`**
5. **Actualizar `.gitignore` para ambos proyectos**
6. **Crear un README principal del monorepo**
